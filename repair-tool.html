Build Conflict Repair Prototype

Based on our catalogue of build conflicts, we developed a Build Conflict Repair Prototype, which can identify and recommend fixes for Unavailable Symbol build conflicts caused by deleted local variables.

Once the tool verifies a build conflict has happened, it uses our scripts to classify the conflict. In case the conflict was caused by Unavailable Symbol of local variables, the tool further investigates the new variable name. After that, the fix recommends the used to update the dangling reference for its nem name. If the developer accepts the recommendation, a new commit is created and the necessary changes are applied.

For future work, we plan to evolve this prototype to cover the remaining build conflict causes of our catalogue. The prototype in available on GitHub. On this page, you can also find there information and additional instructions for using the tool.
